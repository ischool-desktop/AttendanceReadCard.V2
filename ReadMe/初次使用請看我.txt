本文件為穎驊2016/9/1維護此讀卡點名/請假系統整理，希望可以幫助減少後人會遇到的困難，

本公司的讀卡請假系統，使用的是明圃的SR-1800 讀卡機，

使用前必須先在該電腦裝上驅動程式，並把驅動程式中API Library/bin 資料夾內的 OMRAPI.dll、SkCommolo.dll複製到要使用的程式的同一層資料匣(EX:ischool)，

以及將Lib資料夾內的OMRAPI.x32 、OMRAPI.x64 一併複製到輸出的資料夾，

前置作業完成，將SR-1800 讀卡機接上電腦USB 就可以運作了，

讀卡機的基本原理邏輯是在每張讀卡上掃描上百筆格點，然後會回傳每一個格點的筆跡濃度，

比如說第3個格點有濃度2的筆跡，就會得到一個[3,2]的矩陣，

但此筆跡是不是有效畫卡、還是是畫卡者誤塗、擦拭不乾淨，我們就要在程式讀卡系統設定閾值，判定多濃以上是有效的畫卡。
